    - Accept an `onEnded` callback prop, pass to Player's `onEnded` event
    - Wrap Player in a div with: fixed inset-0 z-50 bg-[#0a0a0a] (covers full viewport)

    Update src/app/page.tsx:
    - Add "use client" directive
    - Add state: `const [showIntro, setShowIntro] = useState(true)`
    - Add state: `const [fadeOut, setFadeOut] = useState(false)`
    - When animation ends (onEnded callback):
      1. Set fadeOut=true (triggers CSS opacity transition on the intro wrapper)
      2. After 800ms (transition duration), set showIntro=false (removes intro from DOM)
    - Render structure:
      ```
      {showIntro && (
        <div className={`transition-opacity duration-800 ${fadeOut ? 'opacity-0' : 'opacity-100'}`}>
          <RemotionIntro onEnded={handleAnimationEnd} />
        </div>
      )}
      <main className="min-h-screen bg-[#0a0a0a] text-white flex items-center justify-center">
        <div>
          <h1 className="text-6xl font-bold tracking-tight">crativo</h1>
          <p className="text-xl text-gray-400 mt-4">Creative Software Portfolio</p>
          <p className="text-sm text-gray-600 mt-8">Projects coming soon...</p>
        </div>
      </main>
      ```
    - The main content is always rendered behind the intro overlay, so when intro fades out, content is already there (no flash)

    Important: Use dynamic import for RemotionIntro with `ssr: false` since Remotion Player requires browser APIs:
    `const RemotionIntro = dynamic(() => import('@/components/RemotionIntro'), { ssr: false })`
