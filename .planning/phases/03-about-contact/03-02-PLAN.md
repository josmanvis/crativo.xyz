# Plan 03-02: Contact Section

**Phase:** 3 (About & Contact)
**Created:** 2026-01-24
**Requirements:** CONT-01, CONT-02, CONT-03

## Goal

Build a Contact section that provides multiple ways to connect: email and social links, a working contact form, and calendar scheduling integration.

## Requirements Coverage

- **CONT-01**: Contact section displays email address and social profile links (GitHub, LinkedIn)
- **CONT-02**: Contact section includes working contact form for sending messages
- **CONT-03**: Contact section integrates calendar/scheduling for booking conversations

## Implementation Steps

### 1. Create Contact Data Model

**File:** `src/types/contact.ts`

Define types for:
- `ContactInfo` (email, social links)
- `SocialLink` (platform, url, icon)
- `ContactFormData` (name, email, subject, message)

### 2. Create Contact Data

**File:** `src/data/contact.ts`

Placeholder data including:
- Email address (placeholder: hello@crativo.xyz)
- Social links (GitHub, LinkedIn, Twitter/X)
- Calendar scheduling link (placeholder: calendly.com/placeholder)

### 3. Build Contact Components

**Files:**
- `src/components/contact/ContactInfo.tsx` - Email and social links display
- `src/components/contact/ContactForm.tsx` - Form with client-side validation
- `src/components/contact/CalendarLink.tsx` - Scheduling CTA button

**Design:**
- ContactInfo: Large email link, social icons in row
- ContactForm: Name, email, subject, message fields with proper labels and validation
- CalendarLink: Prominent CTA button with calendar icon

### 4. Create Form Handler

**File:** `src/app/api/contact/route.ts`

API route that:
- Accepts POST requests with form data
- Validates input server-side
- Returns success/error response
- For v1: Log to console (placeholder for email service integration)

### 5. Add Form State Management

**File:** `src/components/contact/ContactForm.tsx` (client component)

Handle:
- Form submission with fetch to API route
- Loading states during submission
- Success/error message display
- Form reset after successful submission

### 6. Create Contact Page

**File:** `src/app/contact/page.tsx`

Server component that:
- Imports contact data
- Composes all Contact components
- Applies consistent layout
- Includes clear CTAs for each contact method

### 7. Add Navigation Link

**File:** `src/components/Navigation.tsx`

Add "Contact" link to site navigation pointing to `/contact`

## Technical Decisions

- **Form Handling**: Client component with Next.js API route (no external service yet)
- **Validation**: Client-side with HTML5 + custom validation, server-side backup
- **Email Service**: Placeholder logging for v1, ready for integration (SendGrid, Resend, etc.)
- **Calendar Integration**: External link to scheduling service (Calendly/Cal.com)
- **Social Icons**: Use heroicons or similar SVG icons
- **Success Feedback**: Toast notification or inline message after submission
- **Component Pattern**: Follow established atomic pattern from Phases 1-2

## Success Criteria

- [ ] `/contact` route renders without errors
- [ ] Email address displays as clickable mailto: link
- [ ] Social links open correct profiles in new tabs
- [ ] Contact form accepts and validates input
- [ ] Form submission triggers API route and shows success message
- [ ] Form displays validation errors for invalid input
- [ ] Calendar link button navigates to scheduling page
- [ ] Navigation includes working link to Contact page
- [ ] Form is accessible with proper labels and keyboard navigation

## Dependencies

- Phase 1 complete (Next.js scaffold, API routes)
- Tailwind CSS configured
- Form validation library optional (can use native HTML5)

## Estimated Complexity

**Medium** - Form handling adds client interactivity and API route, but overall straightforward implementation.
